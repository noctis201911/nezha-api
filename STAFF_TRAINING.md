## 第一课：进入后台

### 你需要知道的三个东西

```
1. 后台网址：https://api.nezha.am/login/admin
2. 系统弹窗密码：用户名 admin，密码 [见内部密码本]
3. 登录账号：上级会单独给你一个员工账号
```

### 进入步骤（图文说明）

![登录页](https://api.nezha.am/admin-docs/screenshots/00_login.png)

**Step 1：** 在浏览器地址栏输入上面的网址，按回车

**Step 2：** 浏览器会弹出一个灰色小窗口（不是网页），输入：
- 用户名：`admin`
- 密码：`[见内部密码本]`
- 点「确定」

**Step 3：** 现在才出现真正的登录页面。输入你的员工账号邮箱和密码，点「登录」

**Step 4：** 进来了！你看到的第一个页面叫「仪表盘」。

## 第三课：你每天最主要的工作——处理线下支付

这是**最重要的日常工作**，一定要学会！

### 什么叫「线下支付」？

顾客下单后，选择了"人民币转账"或"USDT转账"付款方式。  
顾客会把钱转到我们的账户，然后上传转账截图。  
**你的工作：确认收到钱了，点按钮确认。** 

如果你不确认，商家永远不会出餐，顾客就会投诉！

### 怎么操作？

![线下支付列表](https://api.nezha.am/admin-docs/screenshots/06_orders_offline_payment.png)

**第1步：** 点左侧菜单「订单管理」→「线下支付」

**第2步：** 点上方的「待验证」标签，找到待处理的订单

**第3步：** 点那笔订单的单号，进入详情页

**第4步：** 在右侧查看顾客上传的转账截图，核对：
- 转账金额是否和订单金额一致？
- 是否转到了我们的账户？

**第5步：**
- ✅ 没问题 → 点「验证支付」按钮 → 完成！商家开始出餐
- ❌ 有问题 → 点「拒绝」→ 通知顾客重新转账

### 小结

```
打开后台 → 订单管理 → 线下支付 → 待验证 → 逐笔核对 → 点验证支付
                                                           ↑
                                             （每天重复这个动作）
```

## 第四课：处理退款申请

有时候顾客会申请退款，这些申请会发到你这里。

**步骤：**

```
1. 左侧菜单 → 订单管理 → 订单退款 → 新退款申请
2. 查看退款原因
3. 与商家沟通确认情况
4. 决定同意还是拒绝：
   - 同意 → 点「批准」→ 款项退到顾客钱包
   - 拒绝 → 点「拒绝」→ 填写拒绝理由
```

> 📌 **注意：** 退款只能退到顾客的平台钱包，不会直接退到顾客银行卡。  
> 不确定是否要退款时，**先问上级，不要自己决定。**

## 第五课：查看订单状态

### 订单经历的几个阶段

```
顾客下单 → [待处理] → [已接单] → [备餐中] → [配送中] → [已送达]
                                                           
                         ↓ 如果出问题
                      [已取消] 或 [已退款]
```

### 怎么查找某笔订单？

![订单列表](https://api.nezha.am/admin-docs/screenshots/02_orders_all.png)

```
1. 左侧菜单 → 订单管理 → 订单 → 全部
2. 在搜索框输入：订单号 或 顾客手机号
3. 点搜索按钮
```

### 各状态下你能做什么？

| 状态 | 你能做什么 |
|------|-----------|
| 待处理 | 查看是否需要联系商家催接单 |
| 配送中 | 查看骑手位置，遇到超时可在调度中心催促 |
| 已取消 | 查看取消原因，判断是否需要退款 |
| 线下支付 | 核验转账，点「验证支付」 |

## 第六课：遇到异常情况怎么办

### 情况一：有订单卡在「待处理」超过15分钟

```
可能原因：商家没看到新订单通知
处理方式：
1. 联系商家（商家详情页有联系方式）
2. 确认商家在线
3. 如果联系不上，升级给上级处理
```

### 情况二：风控中心有订单等待审核

![风控队列](https://api.nezha.am/admin-docs/screenshots/11_risk_queue.png)

```
什么意思：系统认为这笔订单有点可疑（金额太大/下单太频繁）
处理方式：
- 看一下顾客信息和订单金额
- 这是正常顾客的正常大额订单 → 点「放行」
- 看起来很奇怪 → 不要自己决定，叫上级来看
```

### 情况三：顾客投诉说等了很久没出餐

```
检查步骤：
1. 找到该顾客的订单（搜索手机号）
2. 看订单状态
3. 如果是"线下支付"且没有验证 → 立刻去验证支付
4. 如果已验证但商家还没接单 → 联系商家
5. 如果骑手接了单但没送到 → 在调度中心查看骑手位置
```

## 第七课：你不应该做的事

❌ **不要随意修改业务配置**（尤其是收款账号、佣金比例）  
❌ **不要随意封禁顾客或商家账号**，不确定先问上级  
❌ **不要随意退款**，金额大的退款必须上级审批  
❌ **不要把后台账号密码告诉任何人**  
❌ **不要在业务配置里乱点「清空数据库」**（这会删除所有数据！）  

✅ 每次操作后检查效果  
✅ 遇到没见过的情况先问  
✅ 下班前确认今天的「待验证」线下支付都处理完了
