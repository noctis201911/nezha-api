# 哪吒外卖 — 商家操作与规则手册

> 本手册讲两件事：**平台功能怎么用**（每天的接单、出餐、收款、退款、菜品管理），以及**平台的硬性合规规则（红线）**——红线一经启用即强制执行，请提前知悉并配合。
>
> **平台模式一句话**：顾客的钱**直接付进您本人的收款账户**，平台按单收取佣金、做记录与核验。所以——把每一单做透明、留好痕迹，既是对顾客负责，也是在**保护您自己的账户安全**。

### ⭐ 与美团 / 饿了么最不一样的 6 件事（务必先记住）
这 6 点决定了您在哪吒的操作方式和传统外卖平台完全不同——看懂它们，后面全顺了。

- **钱直接进您自己的账户** —— 顾客付款直付您本人收款账户，平台不代收、也不做 T+1 打款（传统平台是先归集再结算给您）。
- **要您亲自核对到账、再点「确认收款」** —— 钱到没到以**您账户实际到账**为准，确认后订单才出餐（传统平台自动到账，您不用管收款）。
- **退款由您原路退** —— 只能退回顾客**原账户 / 原 USDT 地址**，平台不代扣代退、绝不能退第三方（传统平台由平台代退）。
- **配送您自己叫 Yandex** —— 平台不派骑手（传统平台有平台骑手）。
- **平台只收佣金** —— 从您预存的佣金里按单扣，不抽流水、也不押着您的货款。
- **收款用支付宝 / USDT** —— 没有货到付款（COD）。

# 一、每天都要用（核心）

## 1. 登录商家后台

- **登录网址**：https://api.nezha.am/login/restaurant（如平台另行告知专属网址，以平台告知为准）。
- 使用**邮箱 + 密码**登录（账号必须是邮箱格式）。
- **验证码**：登录页会**直接显示一张图形验证码**（绿色字符图）。把图中字符填进旁边的输入框（不区分大小写），和邮箱、密码一起点「登录」，一步完成、无需重复输入。看不清可点验证码右侧的刷新图标换一张。
- **忘记密码**：点登录页的「忘记密码」，或联系平台帮您重置；重置后请尽快登录并改成您自己的密码。
- **修改密码**：登录后点**右上角您的账户名 / 头像** → 进入「**设置**」页 → 左侧切到「**密码**」→ 填「**新密码 + 确认密码**」→ 点「**保存修改**」。新密码要求：**至少 8 位、含大小写字母 + 数字 + 符号**，且不能是已泄露的常见弱密码。店员账号在各自的「设置 → 密码」页同样可改。（提醒：改密码不需要输入旧密码，所以别在公用电脑上留着登录状态。）
**登录后第一屏**：顶部是「工作台」——一眼看到此刻最该处理的事，不用到处翻。它包含两块：

**① 待办行动**（点任意一张卡 → 直达对应订单列表）
- **待确认收款**：顾客已传付款凭证、等您核对到账后确认的离线单（红色提醒）。
- **待退款**：平台已为顾客取消/退款、等您按原路退还款项的单（黄色提醒）。点进去就是「订单 → 待退款」列表。
- **超时单**：有单卡着太久没处理（红色提醒），尽快处理以免被自动取消。
- **配送催办**：顾客已经在催您分享配送进度、但您还没贴 Yandex 链接时才出现；处理完自动消失（没有就不显示）。
- **差评预警**：一张常驻卡，显示近 7 天「≤3 星差评、且您还没回复」的条数——没有差评时是灰色的 0，有差评时变红。点它直接跳到「客户评价」页，已自动帮您筛好「未回复 + 差评（≤3 星）+ 近 7 天」。左侧「评价」菜单也会显示未回复差评数的红色角标。

> 数字为 0 时卡片会变灰但**不消失**，方便您确认"系统确实没有积压"，而不是"没加载出来"。

**② 今日经营**
- **今日订单**：今天进来的订单总数。
- **今日自收款（已确认到账）**：今天您**已确认收款**的金额合计（在线已付 + 您点过「确认收款」的离线单）。**已上传凭证但您还没确认的单不计入**——钱没确认到账就不算收入，避免误导。旁边带 ≈¥/≈$ 折算。
- **保证金**：显示余额 + **健康状态四档**（充足 / 偏低 / 不足 / 样本不足），**不显示"还能撑几天"这类估算**（数据不足以准确预测，不乱猜）。"不足"会标红提醒"余额不足，可能无法接新单"。
- **店铺评分（累计）**：您的店累计平均星级 + 评价条数（与顾客端看到的一致），还没有评价时显示"暂无评价"。

> 原来的「订单统计 / 年度营业额图 / 热销·好评商品」仍在，只是挪到工作台下方，功能不变。

## 2. 今天 · 作业台 ★（一屏搞定当天所有该处理的单）

**这是您每天用得最多的一页。** 侧栏点「**今天**」进入作业台——把"此刻您该干嘛"全摊在一屏，管理类（改菜 / 对账）不掺进来，省去在十几个订单状态里来回翻。
### 2.1 顶部三件套
- **店态胶囊**：一眼看店当前状态——「营业中」（绿）/「忙碌中」（橙）/「暂停接单」（红）。点它弹出**三档设置**，按您此刻的情况选：
    - **营业中**：正常接单。
    - **忙碌中（仍接单）**：高峰做不过来、但不想关店时用。选个原因（高峰繁忙 / 正在备料 / 人手紧张）+ 填「出餐约需 X 分钟」。顾客在您店铺页顶部会看到一条 🔥「高峰期繁忙，出餐约需 X 分钟」的提示，但**照常能下单**——提前给顾客一个心理预期，能少很多"我的餐怎么还没好"的催促。到点自动回到营业中。
    - **暂停接单**：临时不接单。选「多久后自动恢复」——15 / 30 / 60 分钟，或「不定时（手动恢复）」。暂停后顾客看到您店标「休息中」、**暂时下不了单**；选了时长的还会显示 ☕「约 X 分钟后恢复接单」倒计时，**到点自动恢复，不怕您忙起来忘了开**。**进行中的订单都不受影响。**
  （「暂停接单」与下面「营业时间与临时关店」是同一套状态，在哪一处切都生效。）
- **今日单量 / 自收款**：今天进单数 + 您已确认到账的金额。
- **需动作总览条**：把"要立刻处理"的四类（待确认收款 / 催促·超时 / 退款处理 / 退款申请中）合计给您，数字点进去直达对应订单；全部为 0 时显示「今天没有需要立刻处理的事」。这个合计数字和侧栏「订单」的红色徽标、订单页「需动作」组的数字**永远一致**（同一个来源），不会出现"这里 3、那里 5"对不上的情况。

### 2.2 五张队列卡（卡上直接完成主操作，不用进详情来回翻）
每张卡代表一档"当天要推进的活"，卡上直接给出该单的**主操作按钮**，点了就地完成，不必进详情页：

1. **待确认收款**：顾客传了付款凭证、等您核对到账的离线单。卡上「**确认收款**」进详情核对页（凭证缩略图可点开大图预筛；正式核对仍在详情，不简化步骤）。卡片尾部的灰条是"**还没传凭证**"的单（等顾客传，不算您的待办）。
2. **备餐中**：已接、在做的单。卡上「**出餐 · 叫车**」直接弹出 Yandex 叫车抽屉（和订单列表页同一个）。
3. **配送**：待叫车 + 配送中。待叫车点「**叫车**」；配送中点「**标为已送达**」就地完成。
4. **催促 · 超时**：顾客在催 / 已超时的单（与上面几张卡按单去重、不重复计数），一行一个"跳到该单主操作"。
5. **退款处理**：需要您退款的单，分「**已收款，请退还顾客**」（红）和「**先核对账户，再退还**」（灰）两段（每段标题旁的 ⓘ 可点开看详细说明）。退款一律在详情页按原路核对后标记（**原路退回是平台硬性红线**，卡上不做一键标记）。

> 每张卡最多直显 5 行，超出点「还有 N 单 · 查看全部」进对应筛选。
### 2.3 右栏 + 顶栏铃铛

- **右栏**：差评预警（近 7 天未回复差评）/ 今日经营（单量 · 自收款 · 客单价）/ 快捷入口（全部订单 · 商品 · 一键售罄 · 对账中心）。
- **顶栏铃铛**：超时、顾客催配送这类提醒收在顶栏铃铛里（带数字徽标），点开是列表、可直接跳单——**不会满屏弹窗打扰您**。**新订单仍会响铃**提醒（响铃音量在旁边的喇叭图标里分类可调，见第 5 节）。作业台开着时新单只在队列里高亮 + 响铃、不额外弹浮层；您离开这页时才恢复浮层提醒。
### 2.4 自动刷新与手机模式

- 页面每 **6 秒**自动刷新，新单会自动出现并**高亮**，不用手动刷新。
- 手机上五张队列变成**顶部横滑标签**，点 / 滑切换、默认停在第一个有单的队列；主操作按钮变成整行大按钮，叫车走底部抽屉。
> **小结**：一天的节奏就是——看「需动作总览条」→ 逐张队列卡把单往下推（确认收款 → 备餐 → 叫车 → 送达）→ 有退款/催促单及时处理。作业台开着 + 手机绑好 Telegram（第 5 节），基本不漏单。

## 3. 订单 ★（列表、状态分组、确认收款、下一步操作）

「作业台」帮您抓当天最紧的活；「**订单**」页则是完整的订单台账——查历史、按状态筛、导出对账、逐单操作都在这里。
### 3.1 订单列表怎么看

列表只显示履约需要的信息：**订单号、下单时间、顾客、金额、支付方式、状态、下一步操作**。

- **订单号**本身就是详情链接，点它直接进该单详情页（省去单独的"详情"按钮）。
- **状态**是只读结果；**下一步操作**是可点击按钮（确认收款 / 开始备餐 / 叫车配送 / 标为已送达…），二者分开——您照着「下一步操作」按钮走就行。
- 顾客**手机号默认脱敏**（如 `+374****0001`）；需要联系顾客用左侧「聊天」即可触达，无需手机号。
- **支付方式**列显示顾客本单选择的收款方式（支付宝 / USDT·波场TRC20 / USDT·BEP20）；早期没有付款记录的历史单显示「—」。
### 3.2 状态分组：5 个大组 + 组内二级筛选

订单页顶部不是一排十几个平铺状态，而是按「此刻要不要您处理」分成 **5 个分组标签**，点一个组，下方展开该组的**二级筛选小标签**：
- **需动作**（有单时标红）：需要您现在处理的订单集中在这里——待确认收款、客户催促、超时、退款申请、待退款。这是**横切告警视图**，同一单若还在履约中也会同时出现在「进行中」；组标签上的数字**按订单去重**，是您最该先看的一栏。
- **进行中**：正在履约的订单——待确认收款、已接单、备餐中、待配送、配送中、已预订。
- **售后**：退款相关集中一处——退款申请中、待退款、已退款，便于核对与原路退款。
- **已完结**：已送达、已取消（无未结退款）、支付失败的历史单。**已取消但还挂着未结退款的单只出现在「售后 → 待退款」**，不会同时出现在「已完结 → 已取消」，避免一单两处、账对不上。
- **全部**：真正的全部订单，搜索、导出、「显示已完成 · 近 N 天」筛选都在这里。

> 每个标签旁的数字是该组 / 该状态的实时单数，**与作业台待办、侧栏徽标同一个来源**，各页不会对不上。

**各状态的「下一步操作」按钮**：
- 「待确认收款」→「**确认收款**」（需您确认自己账户到账后才推进；详见 3.3）。
- 「备餐中 / 待配送」的配送单 →「**叫车配送**」（详见 3.5）。
- 「待退款」→「**标记已退款**」（只是声明您已按原路退款并留痕，不触发平台代收代付）。
- 「退款申请中」→「**查看详情处理退款申请**」（避免在列表页误操作）。
- 支付失败 / 已取消 / 已退款 / 已送达等终态 → 显示「**订单已关闭**」。

**「全部」页的三个实用开关**：
- **显示已完成 · 近 N 天**：默认显示全部订单；取消勾选可只看进行中，把已关闭的老单收起；也可填天数只保留最近 N 天的已完成单。只调整**您自己浏览器**里的显示、由本机记住，不改任何数据。
- **今日营收开关**：默认关（遮屏隐私）；打开后在筛选条顶部显示「今日 X 单 · 已确认到账 ֏YYY（≈¥/≈$）」——"已确认到账"= 今日已确认收到款的订单金额合计（排除还没确认收款的待核验单），和首屏「今日经营」卡同源。
- **列设置**：可自选显示哪些列（序号 / 日期 / 顾客 / 金额 / 支付方式 / 更多⋯ / 导出 / 批量打印）；**订单号、订单状态、下一步操作是核心列，始终显示、不可隐藏**。桌面端还能拖动列头右缘调整列宽。都只调整本机显示。
**每行末尾「⋯ 更多」菜单**：把「查看详情」「补打小票」「拒接本单」收在一起，让订单行更清爽。若这张单还是**待确认收款 / 已接单**，菜单最下面会多一个红色的「🚫 拒接本单」——只拒**这一张**订单，点它 → 填拒接原因 → 确认后取消该单并通知顾客（和详情页「无法接单 / 拒单」是同一个动作）。折在菜单里 + 要填原因再确认，双重防误点。
**导出对账**：点列表左上角「**导出**」→ Excel / CSV，导出内容含「支付方式」和「菜品明细」两列，跟随当前筛选（状态 / 搜索 / 日期 / 区域）。导出内容为您自己的订单流水存档。

**手机端**：订单列表自动转为**卡片视图**，每单的核心信息与「下一步操作」按钮就手可及，横向不溢出。
### 3.3 确认收款（顾客付款后您要做的第一件事）

顾客下单并向您的账户转账、上传付款截图后，订单进入「**待确认收款**」等您确认。**这一步只有您确认了，订单才会进入出餐流程**——请及时处理，否则顾客付了钱却一直没动静。
**操作步骤：**
1. 侧栏 →「订单 → 待确认收款」（标题旁红色数字 = 当前待您确认的单数；有新单时后台会语音播报）。
2. 点开订单，在「支付验证」一栏看到**顾客上传的付款截图**和转账信息（金额、单号）。
3. **先去您自己的收款账户（支付宝 / USDT 钱包）核对这笔钱是否真的到账**、金额是否对得上。

### 3.10 长期不确认订单会被自动暂停接单（保护顾客）

为避免顾客不断下单到一家"没人处理"的店，平台设了一道自动保护：

- **什么情况会触发**：近 **2 小时**内您有 **3 单**因超时未处理被系统自动取消（就是上面「3.4 订单超时会被自动处理」里那种"该确认不确认、该接不接"被系统取消的单），**且这 2 小时里您一单都没成功接过**——系统据此判断您可能暂时不在岗，于是自动暂停您接新单，免得顾客继续下单到无人处理的店。
- **在岗但偶尔慢，不会被误伤**：只要这段时间里您还在陆续接单、推进订单（哪怕慢了几单），系统就**不会**暂停您，只会提醒平台关注一下。判断依据是"您这段时间有没有真的处理过单"，不是单纯数超时。
- **只暂停"新"单，别的都不动**：暂停期间顾客下不了新单，但**已接的订单照常进行，平台不碰钱、不扣您的保证金**（这与「4.3 逾期未退款会被自动暂停接单」是两套各自独立的机制）。
- **您会怎么知道**：被暂停时系统给您发**邮件**，同时作业台顶部会出现一条**红色提示条**「已暂停接单」。
- **怎么恢复接单？** 回到岗位后，登录作业台点红条上的「**恢复接单**」按钮即可**立即恢复**（点这个动作本身就说明您在岗了）；也可联系平台运营帮您恢复。**恢复随时能点、没有冷却**——系统不会自动替您恢复，免得把还没回来的店又推给顾客。

## 4. 退款（平台硬性合规要求 · 红线）

平台采用"**顾客直付商家本人账户**"模式，退款由商家执行，但**必须遵守以下硬性规则**，平台会全程记录并核验：

1. **只允许原路退回**：退款只能退回顾客**原始付款的同一账户/渠道**——
   - 人民币（微信/支付宝/银行卡）：退回顾客原付款的同一账户；
   - USDT：只退回顾客**原始付款的钱包地址**（凭交易哈希核验）。
2. **严禁退到第三方**：不得以"退款"名义把钱退给/转给任何非原付款人的账户。这是**反洗钱红线**，违者平台按合规处理（含暂停接单、动用预存佣金、清退）。
3. **金额不超过原订单**：单笔退款金额 ≤ 该订单实付金额。
4. **限额与限笔（防误退/滥退）**：平台对每个商家的退款设有**单笔上限、单日累计金额上限、单日笔数上限**（具体阈值见平台风控设置，可能按商家分级调整）。**超出限额的退款需平台审核后才能放行**，不会被直接拒绝。
5. **USDT 专项**：USDT 仅退回原地址、按链上记录核验；平台不提供 USDT 兑换人民币或提现至中国境内的任何入口。
6. **须填原因 + 留凭证**：每笔退款须注明原因（漏餐/缺货/质量问题等）并上传退款凭证（人民币退款截图 / USDT 退款交易哈希）；相关记录平台留存 **≥ 5 年**。

> **说明**：以上是平台合规底线，目的是保护顾客、也保护您自身账户安全（个人收款无银行拒付保护，规范退款可减少纠纷与账户冻结风险）。

### 4.1 退款完整流程（管理员中心制）

平台采用"管理员中心制"执行退款（商家端暂无自助退款入口）：
1. **顾客在 App 内发起退款申请**（填原因，可附图）→ 进入平台后台。
2. **平台管理员审核**；通过后通知您（商家）按原路退款。
3. **您用本人账户，把钱原路退还给原付款人**：人民币退回原付款账户并**截图保存**；USDT 退回**原始付款的钱包地址**并**记下退款交易哈希**。
4. **把退款凭证交给平台**（经客服/管理员录入后台「退款留痕」）：人民币上传退款截图；USDT 提供退款交易哈希，平台**链上核验**收款地址是否 == 原付款地址、金额是否 ≥ 应退额。
5. 平台留痕该笔退款（含原路锁定地址、链上校验结果），**留存 ≥ 5 年**。

> 🔒 **顾客隐私（脱敏）**：订单详情里顾客的**邮箱与手机号已部分打码**（如 `nezh****t2026@gmail.com` / `+374****0001`），手机号不可直接点击拨打。**如需联系顾客，请用商家后台左侧「聊天」功能**——站内聊天即可触达，无需手机号。

### 4.2 直付单被平台取消/退款后：商家端「待退款」标记流程

一笔直付订单进入「待退款」有两种来源：① 平台管理员取消/退款您已确认收过款的单；② **顾客在 App 自助取消了一笔他已付款给您（已上传付款凭证）的待确认单**——此时钱已到您账户、订单却被取消，需您原路退还。流程：

1. 您会收到推送：「有一笔直付单需要您退款」。
2. 打开**商家后台 → 订单 → 待退款**（侧栏有「待退款」角标），或直接打开该订单详情，看到一张「**待您退款**」卡，含：应退金额（≤ 原订单）、原付款渠道；USDT 单还会显示**须退回的原地址**（系统由原始付款交易反查锁定，只能退这个地址）；顾客邮箱 / 配送电话、付款凭证截图。
3. 您用**本人账户、按原路**把钱退还给原付款人。
4. 退完后回到该卡点「**标记已退款**」：USDT 可选填退款交易哈希（便于平台链上核对），可填备注。确认后该单转为「已退款」留痕。

> ⚠️ 平台**全程不经手这笔钱**——是您在自己的账户里退给顾客；「标记已退款」只是把退款事实记录在平台、便于核对与留痕。

### 4.3 逾期未退款会被自动暂停接单

平台取消/退款一笔直付单后，请**尽快用本人账户把钱原路退还给顾客**，并到「订单 → 待退款」点「标记已退款」。
- **逾期不退的后果（非资金约束）**：超过平台设定时间（默认 4 小时）仍未退款且未标记，系统会**自动催办您（邮件）+ 计入您的风控档案 + 告警平台运营**；继续逾期到平台设定的停接单时长（默认 **36 小时**），系统会**自动暂停您的接单**（只暂停接"新"单，不影响已接订单；停接单与保证金互相独立、不会从保证金扣钱）。
- **怎么恢复接单？** 您用本人账户把钱原路退还顾客、并到「订单 → 待退款」点「标记已退款」后，系统**自动恢复接单**（前提是没有别的逾期未退留痕），无需联系平台。
- **已经退了但忘标记？** 尽快补点「标记已退款」即自动恢复；也可联系平台客服核实解除。

### 4.4 对「待退款」有异议？可发起争议

如果平台生成了一笔「待退款」，但您**核实顾客当初根本没把钱付给您**（凭证伪造/张冠李戴、或链上并未到账），可对这笔待退款**发起争议**，而不是被迫先退一笔您从没收到的钱。
- **怎么发起**：商家后台 →「订单 → 待退款」→ 打开该单的退款核对卡 →「发起争议」，填写核实说明（必填，如"该单顾客未付款、凭证与订单不符"）。
- **发起后**：这笔单进入「争议中」，**逾期催办 / 自动停接单会暂停**，等平台运营核实裁决。
- **平台怎么裁决**：
  - **维持退款义务** —— 平台核实顾客确实付了款，您仍须原路退款；**逾期计时从裁决时刻重新开始**（给您合理处理时间，不拿之前的时间算您"早就逾期"）。
  - **核实未收款** —— 平台核实您确实没收到这笔钱，该单**留痕关闭、您无需退款**。
- 🔴 **红线**：每笔待退款**只能发起一次争议**（裁决即终局）。争议只用于"我确实没收到钱"这类事实争议，**不是拖延退款的手段**。

## 5. 不漏单：新订单提醒（Telegram + 后台提示音 + 手机设置）

> 🔴 **上店前必做（硬性要求）**：**没绑定 Telegram 接单提醒的店，平台激活/上线时会被系统挡住、无法上线**——因为不绑就会漏单（手机锁屏时只有 Telegram 能弹出新单）。确有一台"常开后台的电脑/平板"专门接单的店，可请平台运营给您勾「常开设备豁免」。

### 5.1 绑定 Telegram 接单机器人（推荐，后台自助一步搞定）

1. 登录商家后台 →「业务设置 → 通知设置」，找到顶部「🔔 哪吒接单 / 超时提醒通道」卡片；按 Telegram 那块的提示：手机打开机器人 **@Nz_order_bot**，把卡片上的**验证码**发给它。
2. 回卡片点「**我已发送，检测**」，看到「✓ 已连接」就绑好了。
3. 关联后，每来一笔新订单，机器人会私聊发您一张卡片：单号 / 类型 / 合计 / 商品 / 下单时间。想让多个店员一起收到：把机器人拉进一个店员群，告诉运营用群来接收。

**自己选用哪些方式接收提醒**：在「通知设置」那张卡片里可单独开关「Telegram 提醒」和「邮箱提醒」、自己填收提醒的邮箱。⚠️ 唯一关不掉的是「**订单超时被自动取消、需您原路退款**」这条——为保障顾客，它**始终会发**。

**您会从机器人收到哪些消息（都只跟您自己的店有关，绝不会收到平台内部或别家店的消息）**：
1. 🔔 **新订单**——每来一单发一次（同一单只发一次）。
2. 🔔 **顾客在催配送进度**——提醒您去 Yandex 复制配送追踪链接贴回后台。
3. 🔔 **顾客在催退款 / 有一笔待退款**——该单需您按原路退还，退完去「订单 → 待退款」点「标记已退款」。
4. ✅ **顾客已确认收到退款** / ⚠️ **顾客反映没收到退款**。
5. 💬 **顾客发来的消息 / 咨询**——**您可以直接在 Telegram 里「回复」这条消息**（长按它 → 回复）就能回到顾客，顾客 App 即时收到；也可去后台「聊天」回，两边同步。
6. 📩 **平台回复了您的「问题反馈」**。
7. ⏰ **某笔订单超时还没处理**——卡在待核对凭证 / 待接单 / 备餐阶段太久没动。

### 5.2 后台页面的新单提示音

- 只要您**登录并停留在商家后台**（建议店里用一台电脑 / 平板常开着订单页），系统每约 **6 秒**自动检查一次，有新单会**响一次提示音 + 右下角弹常驻提示条**「🔔 X 个新订单待接单」（直付待核验的单显示「待收款」）；点提示条上的「**立即接单**」会带您直接进入对应订单列表。同一笔订单只响一次。
- 注意：网页提示音依赖后台页面**开着**；把页面 / 浏览器关了就只剩 Telegram 提醒了。
- **调音量 / 只留想要的提示音**：后台顶栏有个 🔊 **喇叭图标**，点开可给每类提示音单独调音量——「新订单 / 订单超时 / 顾客消息 / 平台客服 / 催配送」各一条滑块（0–100%，▶ 可试听），拖到 **0% 就是关掉那一类**；最上面「全部开启」开关可一键全静音。设置存在**这台电脑 / 浏览器**上（每台机器各自设）。**静音只关声音**——提示条和订单列表照常显示，**不会漏单**。首次默认：接单类 90%、消息 / 物流 70%。
### 5.3 📱 开店前必做：把手机这 3 项设对，否则 Telegram 也叫不响您

绑定了机器人，手机设置不对照样收不到声音。开店前逐项检查（安卓 / 苹果都要）：
1. **允许 Telegram 通知**：手机「设置 → 通知 → Telegram」→ 打开「允许通知」，把**声音**和**横幅 / 锁屏显示**都打开。
2. **别让免打扰挡住**：关掉「勿扰模式 / 免打扰 / 专注模式」；若习惯开免打扰，就把 **Telegram 设为「允许例外 / 重点应用」**。
3. **音量开着、别静音**：把**媒体 / 通知音量**调到能听见；店里吵就插个外接小音箱。

**安卓还要多做一步（很容易漏）**：
- 「设置 → 电池 → Telegram」→ 关掉「省电优化 / 后台限制」（允许「无限制后台运行」）。否则手机会为省电把 Telegram 在后台"冻住"，推送被延迟甚至收不到。
- 部分国产手机（小米 / 华为 / OPPO / vivo）还要在「自启动管理」里允许 Telegram 自启动。

**自测（强烈建议开店前做）**：让平台运营或您自己用另一台手机**下一笔测试单**，看**锁屏状态下** Telegram 有没有**弹出 + 响铃**。响了才算配好。

> ⚠️ **「添加到手机桌面」救不了锁屏漏单**：把后台"添加到主屏幕"只是生成一个**网页快捷方式图标**，**不是手机 App**，也没有系统推送能力。手机一锁屏或切到后台，网页提示音就会停。**手机锁屏状态下唯一能弹出新单的，是 Telegram 机器人。**

> 一句话：**绑机器人 = 装了门铃，上面这 3 项 = 给门铃通电。** 只绑不设置，门铃是哑的。

### 5.4 常开设备接单（可选：拿一台设备专门接单，更稳不漏单）

**接单本身不用开任何"模式"**——作业台开着就在收单、会响铃；人在外面靠 Telegram 也能收（5.1）。若店里有条件放一台**专用设备**接单（旧手机 / 平板插电常开，摆在收银台或出餐口）会更省心。作业台右上角点「**新订单提醒设置 ›**」有一页图文指引，照着走即可。要点：

- **屏幕常亮（自动）**：只要停在「今天 · 作业台」页，屏幕**不会自动熄灭**，不用一直去点亮；右下角出现绿色「📱 屏幕常亮」小标就是生效了。（个别旧机型不支持网页常亮，去手机「显示 → 自动锁屏 / 息屏」改「永不」即可。）
- **断连提醒**：万一网络断了、后台收不到新单，页面**顶部会弹红条「⚠️ 后台已断开连接」**提醒您去查网络——网一恢复红条自动消失。这样就不会"以为在接单、其实早掉线漏了单"。
- **仍要开声音**：常开设备也靠响铃提醒，记得音量开着、别静音（音量调节见 5.2）。

> 常开设备和 Telegram 互补：**店里有专用设备常开 → 用它专门接单**（确定这样接单的店，可请平台运营勾「常开设备豁免」代替强制绑 Telegram，见本节开头）；**人在外 / 只有自己手机 → 靠 Telegram 锁屏弹单**。都不是必须"设成某个模式"，按您方便选。

# 二、开店设置（开张必做，之后偶尔调）

## 6. 营业时间与临时关店（开店第一步 · 不设就接不到单）

**入口**：商家后台 → **商家配置（业务设置）→ 餐厅设置**，找到「**营业时间 / Schedule**」一栏，按**周一~周日**逐天设置营业时段。

**怎么设：**
1. 点某一天的「添加时段 / Add Schedule」，填**开始时间 ~ 结束时间**（例：`11:00 ~ 21:00`）。
2. 一天可加**多段**（例：午市 `11:00~14:00` + 晚市 `17:00~21:00`，中间不接单）。
3. 当天不营业 → 不加任何时段，即为**休息日（Offday）**。
4. 删除某段：点该时段上的删除图标。

🔴 **关键规则（务必看）**：顾客**只能在您设置的营业时段内下单**。
- **完全没设营业时间 = 您全天都接不到任何订单**——顾客提交订单时会被拒：「商家已打烊 / Restaurant is closed at order time」，而您后台**不会有任何提示**，很容易误以为"没人下单 / 系统坏了"。
- 某天设为休息日、或当前不在营业时段内 → 顾客同样下不了单。
- **新店开张第一件事就是把营业时间设全**，否则等于没开门营业。

**临时关店（营业状态开关）**：临时有事、备料不足、想提前收摊，可用「营业中 / 暂停营业」开关**手动临时关店**——这**不改**您设置的营业时间，恢复营业时把开关打开即可。（这与作业台的「店态胶囊」是同一套；在作业台的胶囊里还能设「**忙碌中·出餐约需 X 分钟**」和「**暂停 X 分钟后自动恢复**」，详见 §2.1。）

> 💡 **营业时间 vs 临时关店**：营业时间是**每周固定**的常规作息（系统按它自动开关店）；临时关店是**手动的一次性**开关（用完记得打开恢复营业）。

## 9. 起送价 / 最低起订金额

您可以给店铺设一个**起送价**：顾客这一单的商品金额不到这个数就不能下单（避免小额单不划算）。

> 🔔 **平台已给所有店铺设了默认起送价 ֏1100（≈¥20）**。**请登录商家后台核对**这个数对您的店合不合适：嫌高/嫌低都可自己改，改完即时生效；想取消门槛就填 `0`。

1. **怎么填**：商家后台 →「业务设置 / 餐厅设置」→「最低起送金额」字段，以德拉姆 ֏ 计（例填 `1100` ≈ ¥20 起送）；填 `0` 或留空 = 不设门槛。
2. **按什么金额判定**：按顾客这一单的 **券后实付商品金额** = 商品小计 − 商家活动折扣 − 优惠券。**不含**配送费、税、打包费、小费。
3. **顾客看到什么**：不够起送时下单受阻——购物车条显示「还差 ֏X 起送」、「去结算」点不动；用券后才低于起送的，结算页「向商家付款」按钮也会置灰。
4. ⚠️ 起送价**只看商品金额**，配送费顾客另付、不计入判定。

## 10. 上传「分享缩略图」（顾客把您店分享出去时显示的封面图 · 必填）

顾客把您的店铺链接分享到 **Telegram、WhatsApp、Facebook 或外部浏览器** 时，对方看到的不只是网址，还有一张**封面缩略图 + 店名**——这张图就是您后台设的「分享缩略图」，好不好看直接决定别人**愿不愿意点进来**。

> ⚠️ **微信是例外**：直接把链接**粘到微信聊天或朋友圈，微信只显示一行文字网址、不显示缩略图**——这是微信的规则（任何网站链接都这样），不是您设置错了。微信要带图推广见第 17 节。

- **在哪里设**：商家后台 →「店铺设置」→ 页面**最上方第一张卡「分享缩略图 / 搜索信息」**（带红色 `*`）→ 在右侧「分享缩略图」处点击上传 → 拉到底部「保存」。
- **必填规则**：没传过这张图的店，保存店铺设置时会被拦下提示先上传；已经传过的店之后改其它设置不需要重新上传。
- **图片建议**：**横图**最佳，比例约 **1.91:1**（如 **1200×630 像素**），JPG / PNG，≤2MB；放招牌菜特写或干净店面照，少放大段文字（店名系统会自动带上）。
- **没上传会怎样**：系统会自动用您的店铺封面图 / Logo 兜底，分享出去仍有图；但通常不如专门挑一张促销图吸引人，**强烈建议自己传一张**。

# 三、菜品管理

## 12. 给菜加「口味规格 / 加料」（辣度、份量、加蛋… 您自己就能设）

顾客下单时选的「辣度 / 份量 / 加料」这类选项，**您自己在商家后台就能加，不需要找平台超管**。建好保存后，顾客端加购弹窗、购物车、"再来一单"都会自动认到。分两套：

**一、商品规格（同一道菜的不同做法/份量，单选或多选）**
适合：辣度（微辣/中辣/特辣）、份量（小份/大份）、口味（原味/麻辣）等。
- **在哪里设**：「商品列表」→「新增商品」（或在已有菜上点「编辑」）→「**商品规格**」区块 →「添加新规格」。
- **每个规格要填**：规格名（如「辣度」）；「该变体族选择逻辑」选**单选**（顾客只能选一个）或**多选**（能勾多个）；「订单必须项 / 必选」开关（打开 = 顾客**必须选**才能下单，辣度/份量建议打开）；多选时可设「最少 / 最多可选数」；每个「选项名称」+ 该选项的**加价**（不加价填 0；「大份 +500֏」就在该选项加价填 500）。
- 填完拉到底部保存，立即对顾客生效。

**二、加料 / 加项（一批可复用的加项，挂到多道菜上）**
适合：加宽粉、加午餐肉、加香菜… 这种**多道菜都能加**的料。
- **先建加料库**：「加料/加项」→「加料列表」→「新增加料」→ 填名称 + 价格。
- **再挂到菜上**：在「新增/编辑商品」页的加料选择里勾上这道菜允许的加料。
- 区别：规格是"这道菜本身的做法"（单道菜独有）；加料是"一批通用加项"（建一次、多道菜复用）。

**⚠️ 最容易踩的坑**：
- **该必选的没设必选** → 顾客不选辣度就下单了。辣度 / 份量务必打开「必选」。
- **单选 / 多选选反** → 一道菜只能选一种的用「单选」。
- **加价填错** → 加价是"在菜价基础上加多少֏"，不加价填 0。
- 改完务必**保存**，保存后顾客端会自动同步，无需平台介入。

## 13. 调整菜品在顾客端的「陈列顺序」（拖拽排序）

想让招牌菜、主推菜排在菜单前面？进「菜品」列表页，点右上角「**菜品排序**」按钮：
1. 选一个分类（下拉框，按您店里的分类分组）。
2. 按住任意一道菜、上下拖动，调整它在该分类里的位置。
3. **松手即自动保存**，页面提示「已保存」。

小提示：每个分类单独排、互不影响；新加的菜默认排在所在分类的**最后**，想置顶就拖上来；顾客若自己切了「快送 / A→Z」等排序，会按他选的规则显示，您排的是「默认」档下的展示序。

## 14. 一键「今日售罄」（某道菜今天卖完了，次日自动恢复）

某道菜今天备料卖完、但明天还做？不用彻底下架——进「菜品」列表页，打开那一行的「**今日售罄**」开关：
- 开关一开，顾客端这道菜立刻变**灰**、盖上「已售罄」，顾客加不了购；但**菜品仍然可见**（不是隐藏），让顾客知道明天还有。
- **次日自动恢复**可售——不用第二天再去开；也可随时手动关掉立即恢复。对**不限量**的菜也有效。

**和「上下架」的区别**（别用错）：「上下架」= 把菜从菜单里**彻底隐藏**（长期不卖了才用）；「今日售罄」= 菜**还在菜单上**、只是今天暂时买不了、**第二天自动回来**（对标美团/饿了么的「沽清」）。日常"今天卖完了"就用这个开关。

## 15. 批量操作菜品（一次改多道菜：上下架 / 改价 / 改分类 / 删除）

菜品多、要一次性调整时不用一行行点。进「菜品」列表页，每行左侧有**勾选框**，表头有**全选**：
1. **勾选**要操作的菜，页面**底部浮出操作栏**显示「已选 N 项」。
2. 在操作栏选一个动作：
   - **上架 / 下架**：一次性全部上/下架（下架的菜会同时从顾客购物车移除）。
   - **改价**：**设折扣**（按百分比或固定金额统一设折扣，各自原价不变；固定折扣 ≥ 某菜原价会自动跳过）；或**按比例调价**（原价统一 ×百分比，涨价填 `10`、降价填 `-20`）。
   - **改分类**：统一归到某个分类（可到二级子分类）。
   - **删除**：**有历史订单的菜删不掉**（会自动跳过并提示）——它们关系到销量报表和复购，不想卖请用「下架」。
- 一次最多操作 200 道菜。

## 16. 给顾客发「店铺公告」（营业提醒 / 出餐时长 / 临时通知）

您可以在店头挂一条**公告条**，顾客一进餐厅页就能看到（店名下方、暖黄色那条），适合发：营业/休息时间、高峰期出餐要等多久、临时停售某道菜、节日通知等。
- **在哪里填**：商家后台 →「店铺信息 / Shop Info」页 →「公告 / Announcement」：① 先把「公告开关」打开；② 在「公告内容」框写要说的话（最多约 250 字）；③ 保存。
- **关掉公告**：把公告开关关掉即可，顾客端立刻不再显示。
- ⚠️ 公告只发**真实信息**（营业/出餐/通知），别拿它发虚假促销。高峰期建议写明出餐时长，减少顾客催单。

> 另：餐厅页会自动显示「本店热销榜 TOP1/2/3」——按您店**近 30 天真实销量**自动排，**您不用设置、也改不了排名**（防刷榜）。新店或还没出过单时暂不显示。

## 17. 把店铺 / 菜品分享出去（推广就靠这步）

**怎么把您的店分享出去**：用手机打开**您自己的店铺页**，点店铺顶部大图**右上角的「分享」图标**（放大镜🔍旁边、一个向上箭头）→ 弹出分享窗，分两块：**发微信**点「分享海报到微信」；**普通分享**（Telegram / WhatsApp / 复制链接）发链接给好友。

**📱 微信怎么带图推广（重要，用「推广海报」）**：微信里粘链接只显示一行文字、不带图（微信规则）。所以我们做了「**推广海报**」——一张带图和二维码的图片，图片在微信一定显示，好友长按识别二维码就进您的店：
- **店铺海报**：手机进自己的店铺页 → 点顶部大图右上角分享图标 →「分享海报到微信」→ 选微信发出（不弹面板时长按海报图保存再发）。

- **想要链接自动出带图卡片**：在 Telegram / WhatsApp 里直接发链接即可（这些 App 会自动显示带图卡片）。

**顾客也能单独分享您的某道菜（商品图 = 免费推广）**：顾客打开任意一道菜的详情弹窗，可点右上角分享按钮把**这一道菜**单独发给朋友或群里，对方看到的卡片显示**这道菜的商品图 + 菜名 + 价格**，点开直接跳到您的店并自动弹出加购窗。所以**每道菜的图都尽量拍清楚、好看**（实拍、光线足、主体突出），这是不花钱的获客。

## 18. 对账中心（预存佣金 / 广告 / 押金对账 + 余额不足提醒）

**入口**：商家后台左侧栏「对账中心」（`/restaurant-panel/nezha-deposit`）。

**这是什么**：平台不碰您的货款（顾客的钱直接进您账户）。平台收入靠向您收佣金——您先把一笔「**预存佣金**」交给平台，平台按每单自动从中扣佣。**余额降到 0 或为负时，您的店会自动暂停接收新订单**（扣佣开关开启后生效），充值到大于 0 后自动恢复。此时**顾客端会看到您的店显示「休息中」**——和正常打烊一样，顾客看不到报错、也不知道是余额原因。所以请留意余额、及时充值。

**您能看到**：
- 当前预存佣金余额（**若为负，红色显示「当前欠款」**）。
- **顶部三账户总览**：预存佣金余额、广告余额、押金（可退质押账户）。
- **账户切换 + 对账**：切到「预存佣金」或「广告」，各自可按日期区间（本月 / 近30天 / 自定义）筛选，看 期初余额 / 本期充值 / 本期扣佣 / 退款返还 / 期末余额 汇总 + 该账户流水；两个账户各自独立、互不混算。
- **导出对账单**：点「导出 Excel」或「导出 CSV」下载存档。

**余额不足邮件提醒（您可自助设置）**：① 打开「开启余额不足邮件提醒」开关；② 设「当余额低于」多少（**单位为德拉姆 ֏**，输入框旁实时显示 ≈¥/≈$）；③ 填提醒邮箱（可与注册邮箱不同）；④ 保存。系统每天检查一次，低于阈值（或为负）给该邮箱发提醒，**每天最多一封**。

**怎么充值（自助申请充值 · 平台开放后可用）**：在对账中心页点「**申请充值**」——① 按页面显示的**平台收款账户**（支付宝 / 银行，**不收 USDT**）转账；② 填转账金额 + 上传转账凭证图（必填）+ 备注（选填）→ 提交，状态显示「审核中」（未审核可撤回）；③ 平台核对到账后为您入账，余额即时增加。以**平台实际到账金额**入账（可与您填的申请额略有出入，如手续费 / 汇率差异）。预存佣金是您预付给平台的佣金（B2B），与顾客货款无关。

> **审核结果怎么通知您**：入账 / 打回（以及押金退款的「已审批 / 已放款 / 被打回」）会通过 **Telegram（若已绑商家 TG）+ 商家后台顶栏「待办提醒」🔔铃铛**告诉您，点铃铛条目直接进对账中心。

**申请退回押金（营业中退回多缴 / 超额押金 · 平台开放后可用）**：如押金多缴了、或应缴档下调有富余，可在对账中心申请退回**超出最低留存**的部分。规则（平台硬性红线）：**只退回您 KYC 核验的本人法币账户**（收款账户锁定 KYC 收款户名，不可临时改），**严禁退到第三方账户**；只退超出最低留存押金的部分（要全额退/准备离场请走「退出结算」）；平台核实后**线下把款转回您的账户**；您欠平台佣金时不予退押。退押金为低频动作，有频率限。

## 19. 投广告给自己店买曝光

平台提供两种付费推广，都**从与货款完全分开的账户扣费**，不会动您的货款。

**A. 广告（按天买曝光 · 默认免费）**
给自己的店投广告买更多曝光（在顾客端「综合排序」里把您的店往前提）。**平台没开收费前，投广告不花钱**。开启收费后的规则：
1. **按天算**：广告费 = 平台单价（如 1000 ֏/天）× 投放天数。提交时显示「预估总费用」和「当前保证金余额」。
2. **从哪扣**：从您的**预存佣金 / 保证金**里扣。
3. **什么时候扣**：审核通过时不扣、还能免费取消；到**投放起始日正式上线那一刻**才扣全额。提交页有一句必须勾选的确认：广告将在有效期起始日正式投放时扣费，**扣费后不可取消、不可退款**（平台主动下架除外）。
4. **余额不够**：到起始日保证金不够 → 广告不上线 + 系统发邮件提醒充值（不赊账、不自动重试）。
5. **平台主动下架才会退**：平台/超管强制下架已扣费的广告，按还没投放的天数比例退；您自己中途停投不退。

**B. 竞价推广（按点击付费 · 平台开启后可见）**
让您花钱把店在顾客的餐厅列表里排得更靠前。
- **按"点击"收费**——只有真实顾客点进您的店才扣钱，光展示不点不收费；刷单 / 自己点不算。
- **三个设置**：推广开关（随时可关）；想多靠前（低 / 中 / 高，档位越高越靠前、单次点击越贵）；每天最多花（每日上限，当天花到自动停、次日重置）。
- **广告余额**：推广花的是"广告余额"，和经营保证金**完全分开**——推广烧钱不会动保证金、不会把店买下线。广告余额由平台按对公付款充值。
- **看效果**：面板有"今日已花 / 今日点击"。
- **平台红线**：付费靠前**不改变**店铺真实评分 / 销量的展示，也不贴"热门 / 精选 / 评分最高"等标签。

# 五、沟通与帮助

## 20. 聊天（顾客消息 / 订单卡片 / 客服）

顾客在 App 里联系您时，可以**一键把某一笔订单作为卡片发给您**。您在商家后台「聊天」里会看到一张订单卡片，含订单号、当前状态、取餐号、金额、下单时间，点卡片可直接跳到该订单详情页。**好处**：顾客同时有好几笔单时，您一眼就知道他在问哪一单。

**新消息提醒**：您开着后台时，顾客发来新消息会**自动响铃 + 右上角弹提示**，会话列表和当前对话也会**自动刷新**（约每 10 多秒检查一次）。

## 28. 商家服务规范与纠纷处理细则

### 一、每一单的基本服务要求（履约规范）
1. **及时接单、及时确认收款**：别让单子干等。
2. **如实填写预计出餐时间**：填的分钟数要贴近真实。
3. **按时出餐、及时推进订单状态**：出好餐就标「配送中」，别让状态卡着。

5. **保持沟通畅通**：开着后台、听到响铃及时回复顾客与平台客服。
6. **规范退款**：该退的钱一律**原路退回顾客本人账户**，并留好凭证。

### 二、订单超时怎么处理（要点摘要）
- **迟迟不确认收款／不接单**：满 10 分钟系统邮件催您；满 20 分钟自动取消该单。**若顾客已付款，您必须按原路退还本人**。
- **备餐超时**：会标「备餐异常」并催您，但**不会**自动取消。
- 所有超时的**自动取消 ≠ 平台替您退款**——退款始终由您原路退给顾客本人。

### 三、顾客投诉／申诉怎么处理
1. **先核实**：这一单送到没有、地址电话对不对、Yandex 是否真送达、餐品是否有问题。
2. **该退就退**：确属您的责任，用顾客原付款方式原路退回本人账户。
3. **留好证据**：把配送记录、站内聊天记录、退款截图／USDT 退款哈希留存，交平台留痕。
4. **告知平台**：处理完配合平台运营核对。

> 平台对争议单设有**人工裁决**通道：顾客提交「没收到退款」争议后，平台超管会介入核对、依据记录判断（不自动偏袒任何一方，也防顾客诬告）。

### 四、⭐ 配送物流信息与「纠纷判断依据」（重点）
**平台强烈建议：每一单安排配送后，都在订单详情「配送链接」栏贴上 Yandex Go 的「分享配送行程」链接**（必须是 yandex.ru／yandex.com 官方域）。贴上后订单自动推进到「配送中」，顾客能实时看到餐到哪了。

**为什么如此重要——它直接关系到出纠纷时谁占理：**
- ✅ **分享了**：配送全程可追溯；万一起纠纷（「我没收到」「送太慢」），**平台调取记录时您有据可查、对您有利**。

**平台在纠纷时依据的客观留存记录**包括：订单状态时间线、是否上传／分享了配送物流信息、站内聊天记录、收款／退款凭证、顾客申诉记录等。据此平台会判断责任在谁、决定管理措施、指导退款责任归属（但退款执行始终由您原路完成）。

> 🔴 **务必记住**：**是否主动留痕、是否分享配送物流信息，就是您在这种判断里最有力（或最缺失）的证据。** 养成每单分享的习惯，出纠纷时吃亏的不会是您。

> 适用于已入驻平台「本地生活」板块的服务型商户（移民 / 签证 / 美容美发 / 按摩 / 包车等）。此面板与外卖餐厅商家后台不同，**入口是** `api.nezha.am/m/login`。

**开通与登录**
1. 联系平台开通账号，平台会往您邮箱发一封「设置登录密码」邮件。
2. 点邮件里的链接设置密码（24 小时内有效），然后用**邮箱 + 密码**登录 `api.nezha.am/m/login`。
3. 忘记密码：登录页点「忘记密码 / 首次设置密码」，输邮箱自助重置。

**维护店铺信息**
- 登录后可预览店铺当前展示（顾客看到的样子），点「编辑店铺信息」修改：店铺介绍、服务项、相册、Logo、联系方式（电话 / WhatsApp / Telegram / 微信）、营业时间、到店优惠。

## 商家页笔记（图文动态 / 作品展示）〔2026-07-10 · 默认关，上线由平台统一开〕

你可以在商户自助管理面（`api.nezha.am/m`）发布「笔记」——图文形式的店内动态、作品、招牌展示。顾客在你的店铺页也能写下他们的体验。

**怎么发**：登录 `/m` →「笔记」→「写新笔记」→ 传 1–9 张图（至少 1 张）+ 选填标题 + 写正文 → 提交审核。平台审核通过后，笔记会显示在你的店铺页「笔记」卡。

**红线（违反会被驳回或下架）**：
- **不许写联系方式**：笔记里不要放电话、微信、Telegram、网址等。顾客要联系你，走店铺的「联系方式」卡。

**说明**：笔记不是评分体系（没有星级、点赞、好评率）。是否在顾客端展示由平台统一开关控制；你发的笔记即使暂未展示，也会保留在 `/m` 里，通过审核后自动上架。
