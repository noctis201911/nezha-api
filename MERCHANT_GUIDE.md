# 哪吒外卖 — 商家规则与操作手册 (Merchant Guide)

> 本手册告知商家须遵守的平台规则与操作方式。
> **状态说明**：按平台文档约定，具体"操作步骤"随对应功能上线后补全；本期先明确**平台硬性合规规则（红线）**，这些规则一经上线即强制执行，请商家提前知悉并配合。

## 退款规则（平台硬性合规要求 · 红线）

平台采用"**顾客直付商家本人账户**"模式，平台不经手资金。退款由商家执行，但**必须遵守以下硬性规则**，平台会全程记录并核验：

1. **只允许原路退回**：退款只能退回顾客**原始付款的同一账户/渠道**——
   - 人民币（微信/支付宝/银行卡）：退回顾客原付款的同一账户；
   - USDT：只退回顾客**原始付款的钱包地址**（凭交易哈希核验）。
2. **严禁退到第三方**：不得以"退款"名义把钱退给/转给任何非原付款人的账户。这是反洗钱红线，违者平台按合规处理（含暂停接单、动用预存佣金、清退）。
3. **金额不超过原订单**：单笔退款金额 ≤ 该订单实付金额。
4. **限额与限笔（防误退/滥退）**：平台对每个商家的退款设有**单笔上限、单日累计金额上限、单日笔数上限**（具体阈值见平台风控设置，可能按商家分级调整）。**超出限额的退款需平台审核后才能放行**，不会被直接拒绝。
5. **USDT 专项**：USDT 仅退回原地址、按链上记录核验；平台不提供 USDT 兑换人民币或提现至中国境内的任何入口。
6. **须填原因 + 留凭证**：每笔退款须注明原因（如漏餐/缺货/质量问题等）并上传退款凭证（人民币退款截图 / USDT 退款交易哈希）；相关记录平台留存 ≥ 5 年。

> **说明**：以上为平台合规底线，目的是保护顾客、也保护商家自身账户安全（个人收款无银行拒付保护，规范退款可减少纠纷与账户冻结风险）。

### 退款操作流程（功能已上线 · 商家配合步骤）

平台采用"管理员中心制"执行退款（商家端暂无自助退款入口）。一次退款的完整流程：

1. **顾客在 App 内发起退款申请**（填写原因，可附图）→ 进入平台后台。
2. **平台管理员审核**该申请；通过后会通知你（商家）按原路退款。
3. **你（商家）用本人账户，把钱原路退还给原付款人**：
   - 人民币：退回顾客**原付款的同一微信/支付宝/银行账户**（即付款截图上的付款人），**截图保存**；
   - USDT：退回顾客**原始付款的钱包地址**（即你当初收款那笔的 from 地址），**记下退款交易哈希**。
4. **把退款凭证交给平台**（经客服/管理员录入后台「退款留痕」）：
   - 人民币：上传**退款截图**；
   - USDT：提供**退款交易哈希**，平台会**链上核验**该笔退款的收款地址是否==原付款地址、金额是否≥应退额。
5. 平台留痕该笔退款（含原路锁定地址、链上校验结果），**留存 ≥5年**。

> ⚠️ **务必原路、务必留证**：USDT 退到非原始地址、或金额/地址对不上，链上核验会判**失败**；人民币退给非原付款人属**退第三方**（反洗钱红线）。超出平台限额的退款会先转平台审核、放行后再退。如有疑问先联系平台客服，不要自行退给第三方账户。

## 钱包（侧栏「私域资金池」）— 说明

哪吒采用"**顾客直付商家本人账户**"模式，平台**不向商家打款**。因此：

1. **钱包没有"提现/打款"功能**：原系统残留的「提现申请 / 请求提现 / 待提现 / 已提现总额 / 可提现余额 / 下次打款(Next Payouts)」等入口**已隐藏**，你不会也不需要在平台申请提现——你的营业款由顾客直接付到你自己的收款账户。
2. **钱包页当前展示**：
   - **手持现金**：货到付款(COD)等以现金/直收方式收到、需与平台结算口径对账的金额；
   - **待付余额（应付平台）**：你需要支付给平台的金额（如佣金；扣佣机制上线后启用）。金额大于 0 时，会出现「立即支付 / Pay Now」入口供你向平台付款；
   - **累计佣金**：历史佣金统计（直付订单暂不计入）；
   - **支付记录**：你向平台付款的历史明细。
3. 平台仅靠**佣金**收费（从你预存的**预存佣金**余额中按单扣除），不经手你的营业资金。

> 说明：扣佣/充值台账等功能仍在建设中，上线后本节会同步更新具体操作步骤。

## 菜价与收款币种（德拉姆 ֏ 计价 · 顾客付人民币/USDT）

1. **你按德拉姆（֏）录菜价**：后台所有菜价、金额都以亚美尼亚德拉姆 ֏ 为准（例：一份兰州拉面填 `1500` 即 ֏1500）。
2. **顾客看到换算价**：顾客端 App 在 ֏ 主价旁显示换算后的 **人民币(¥) / 美元($)**——浏览/菜单页是"≈"参考价，**结算页是精确"应付 ¥X / X USDT"**，顾客照这个金额付给你。
3. **汇率由平台统一维护（你不能改）**：换算汇率是平台全局设定，也是**事实结算汇率**——顾客付的 ¥/USDT = 你菜价 ֏ 总额 ÷ 平台汇率。
4. ⚠️ **汇率波动由你承担**：你按 ֏ 标价却收 ¥/USDT，市场汇率与平台汇率有偏差时会轻微多收/少收；平台会尽量贴市场维护，偏差较大请及时联系平台。
5. **你实收 = 顾客实付**：平台不经手资金，结算页显示的"应付 ¥/USDT"就是顾客打到你账户的金额，请按此核对收款。

## 收款码（支付宝 / 微信 / USDT）— 你要提供什么 〔2026-06-15 微信独立成单独码〕

顾客在结算页可选 **支付宝 / 微信 / USDT** 三种方式付款，钱**直接付进你本人的账户**，平台不经手。你需要把本人收款码/地址提供给平台录入（目前由平台运营代录，不在商家后台自助上传）。

1. **支付宝收款码**：本人支付宝「收款码」二维码截图 + 收款人姓名（顾客转账时核对用）。
2. **微信收款码**：本人微信「收款码」二维码截图。⚠️ **微信码与支付宝码是两张分开的码**，必须都提供——
   - 此前平台用一张码同时给微信/支付宝扫，现已拆开：顾客选「微信」只会看到你的**微信**码、选「支付宝」只看到**支付宝**码。
   - **没提供微信码 = 顾客无法用微信付你的款**（结算页会显示「本店暂未设置微信收款码」）。建议两张都给齐。
3. **USDT 收款地址**：钱包地址 + 网络（TRC20 或 BSC），按需提供。

> 💡 **USDT 地址会自动生成二维码**：顾客在结算页/订单页可直接扫这张由你地址自动生成的二维码转账（无需你额外上传图，二维码永远跟地址一致）。⚠️ **地址务必填对、逐字核对**——USDT 转账不可逆，地址错一位，顾客的钱会永久丢失且无法追回。

> **顾客怎么用微信付**：顾客在手机浏览器里**长按你的微信码保存到相册 → 打开微信「扫一扫」→ 从相册选这张码**完成付款（手机浏览器无法一键跳转微信，这是微信平台限制）。所以请提供**清晰、可被微信识别**的收款码截图。
>
> 🔴 **合规提醒**：收款码必须是**你本人**的真实收款账户。平台不归集资金，顾客付款全程进你账户；收款码图片在平台侧加密存储。收到款后请按结算页显示的「应付 ¥/USDT」金额核对。
