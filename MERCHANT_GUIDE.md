## 营业时间与临时关店（开店第一步 · 不设就接不到单）〔2026-06-17 补充〕

**入口**：商家后台 → **商家配置（业务设置）→ 餐厅设置**，找到「**营业时间 / Schedule**」一栏，按**周一~周日**逐天设置营业时段。

**怎么设：**
1. 点某一天的「**添加时段 / Add Schedule**」，填**开始时间 ~ 结束时间**（例：`11:00 ~ 21:00`）。
2. 一天可加**多段**（例：午市 `11:00~14:00` + 晚市 `17:00~21:00`，中间不接单）。
3. 当天不营业 → 不加任何时段，即为**休息日（Offday）**。
4. 删除某段：点该时段上的删除图标。

🔴 **关键规则（务必看）**：顾客**只能在你设置的营业时段内下单**。
- **完全没设营业时间 = 你全天都接不到任何订单**——顾客点进你的店、提交订单时会被拒：「**商家已打烊 / Restaurant is closed at order time**」，而你后台**不会有任何提示**，很容易误以为"没人下单 / 系统坏了"。
- 某天设为休息日、或当前不在营业时段内 → 该时段顾客同样下不了单。
- **新店开张第一件事就是把营业时间设全**，否则等于没开门营业。

**临时关店（营业状态开关）**：临时有事、备料不足、想提前收摊，可在商家后台用「**营业中 / 暂停营业**」开关**手动临时关店**——这**不改**你设置的营业时间，恢复营业时把开关打开即可继续接单。适合"今天提前打烊""临时缺货"等突发情况，比改营业时间方便。

> 💡 **营业时间 vs 临时关店**：营业时间是**每周固定**的常规作息（系统按它自动开关店）；临时关店是**手动的一次性**开关（用完记得打开恢复营业）。常规作息用营业时间设好，突发情况用临时开关，两者配合。

## 设置店铺所属类目（让顾客在首页分类里找到你）〔2026-06-17 上线〕
顾客首页搜索框下面那排"类目"（如 槟榔零食 / 超市百货 / 川渝香菜 / 南方菜系 / 粥面包点），顾客点进某个类目，只会看到**勾选了该类目**的店。所以**开店时一定要选对自己的类目**，否则顾客从类目里找不到你。
- **在哪里设**：商家后台 → 「店铺设置 / Restaurant Settings」→ 找到「**菜系 / Cuisine**」多选框 → 勾选你店所属的一个或多个类目 → 拉到页面底部「保存」。
- **可多选**：比如一家又做川菜又做江浙菜的店，可同时勾「川渝香菜」「南方菜系」。
- **找不到合适的类目？** 类目由平台统一维护。需要新增类目请联系平台运营，平台在后台加好后，你这边的下拉框里就会自动出现。
- 选完保存后，顾客在首页对应类目里就能看到你的店。

## 上传「分享缩略图」（顾客把你店分享出去时显示的封面图 · 必填）〔2026-06-22 上线〕
顾客把你的店铺链接分享到**微信、Telegram、WhatsApp、朋友圈**等地方时，对方看到的不只是一条网址，还有一张**封面缩略图 + 店名**。这张图就是你后台设的「**分享缩略图**」。图好不好看，直接决定别人**愿不愿意点进来**——它是你免费的推广海报。

- **在哪里设**：商家后台 →「店铺设置 / Restaurant Settings」→ 页面**最上方第一张卡「分享缩略图 / 搜索信息」**（带红色 `*`）→ 在右侧「分享缩略图」处点击上传 → 拉到底部「保存」。
- **必填规则**：没传过这张图的店，保存店铺设置时会被拦下并提示先上传；**已经传过的店**之后改其它设置**不需要重新上传**，不会打扰你。
- **图片建议**：
  - **横图**最佳，比例约 **1.91:1**（如 **1200×630 像素**），JPG / PNG，≤2MB。
  - 放**招牌菜特写**或**干净的店面/门头照**，画面清晰、主体突出。
  - **少放大段文字**（分享卡片上字会被缩小看不清），店名系统会自动带上，不用你在图里写。
- **没上传会怎样**：系统会自动用你的**店铺封面图 / Logo** 兜底，分享出去仍有图、不会是空白；但那通常不如你专门挑一张促销图吸引人。所以**强烈建议自己传一张**。
- 💡 小提示：微信里分享有时缩略图出现较慢或偶尔不显示，属各平台抓取差异，多为正常现象。

## 给菜加「口味规格 / 加料」（辣度、份量、加蛋… 你自己就能设，不用找平台）〔2026-06-20 上线〕

顾客下单时选的「辣度 / 份量 / 加料」这类选项，**你自己在商家后台就能加，不需要找平台超管**。建好保存后，顾客端加购弹窗、购物车、"再来一单"都会自动认到。分两套，按需要用：

### 一、商品规格（同一道菜的不同做法/份量，单选或多选）
适合：辣度（微辣/中辣/特辣）、份量（小份/大份）、口味（原味/麻辣）等。
- **在哪里设**：商家后台 →「**商品列表**」→「**新增商品**」（或在已有菜上点「编辑」）→ 找到「**商品规格**」区块 → 点「**添加新规格**」。
- **每个规格要填**：
  - 规格名（如「辣度」「份量」）。
  - 「**该变体族选择逻辑**」：选「**单选**」= 顾客只能选一个（辣度、份量用这个）；选「**多选**」= 顾客能勾多个（"加料式"选项用这个）。
  - 「**订单必须项 / 必选**」开关：打开 = 顾客**必须选**才能下单（辣度、份量建议打开）；关闭 = 选不选都行。
  - 多选时还可设「最少 / 最多可选数」（如加料最多选 2 个）。
  - 每个「**选项名称**」+ 该选项的**加价**（不加价填 0；如「大份 +500֏」就在该选项加价填 500）。
- 填完拉到页面底部保存，立即对顾客生效。

### 二、加料 / 加项（一批可复用的加项，挂到多道菜上）
适合：加宽粉、加午餐肉、加金针菇、加香菜… 这种**多道菜都能加**的料。
- **先建加料库**：商家后台 →「**加料/加项**」→「**加料列表**」→「**新增加料**」→ 填名称 + 价格（如「加香菜 +200֏」）。
- **再挂到菜上**：在「新增/编辑商品」页的加料选择里，勾上这道菜允许的加料。
- 和「商品规格」的区别：规格是"这道菜本身的做法"（单道菜独有）；加料是"一批通用加项"（建一次、多道菜复用）。"加料式选项"用上面的多选规格做、还是用这套独立加料做都行，顺手哪个用哪个。

### ⚠️ 设置时最容易踩的坑
- **该必选的没设必选** → 顾客不选辣度就下单了，你不知道做几分辣。辣度 / 份量这类**务必打开「必选」**。
- **单选 / 多选选反** → 辣度若设成「多选」，顾客会同时勾"微辣+特辣"。一道菜只能选一种的，用「单选」。
- **加价填错** → 加价是"在菜价基础上**加**多少֏"，不加价填 0，别填成整道菜的总价。
- 改完务必**保存**；保存后顾客端加购、购物车、"再来一单"会自动同步，**无需平台或超管介入**。

## 退款规则（平台硬性合规要求 · 红线）

平台采用"**顾客直付商家本人账户**"模式，平台不经手资金。退款由商家执行，但**必须遵守以下硬性规则**，平台会全程记录并核验：

1. **只允许原路退回**：退款只能退回顾客**原始付款的同一账户/渠道**——
   - 人民币（微信/支付宝/银行卡）：退回顾客原付款的同一账户；
   - USDT：只退回顾客**原始付款的钱包地址**（凭交易哈希核验）。
2. **严禁退到第三方**：不得以"退款"名义把钱退给/转给任何非原付款人的账户。这是反洗钱红线，违者平台按合规处理（含暂停接单、动用预存佣金、清退）。
3. **金额不超过原订单**：单笔退款金额 ≤ 该订单实付金额。
4. **限额与限笔（防误退/滥退）**：平台对每个商家的退款设有**单笔上限、单日累计金额上限、单日笔数上限**（具体阈值见平台风控设置，可能按商家分级调整）。**超出限额的退款需平台审核后才能放行**，不会被直接拒绝。
5. **USDT 专项**：USDT 仅退回原地址、按链上记录核验；平台不提供 USDT 兑换人民币或提现至中国境内的任何入口。
6. **须填原因 + 留凭证**：每笔退款须注明原因（如漏餐/缺货/质量问题等）并上传退款凭证（人民币退款截图 / USDT 退款交易哈希）；相关记录平台留存 ≥ 5 年。

> **说明**：以上为平台合规底线，目的是保护顾客、也保护商家自身账户安全（个人收款无银行拒付保护，规范退款可减少纠纷与账户冻结风险）。

### 退款操作流程（功能已上线 · 商家配合步骤）

平台采用"管理员中心制"执行退款（商家端暂无自助退款入口）。一次退款的完整流程：

1. **顾客在 App 内发起退款申请**（填写原因，可附图）→ 进入平台后台。
2. **平台管理员审核**该申请；通过后会通知你（商家）按原路退款。
3. **你（商家）用本人账户，把钱原路退还给原付款人**：
   - 人民币：退回顾客**原付款的同一微信/支付宝/银行账户**（即付款截图上的付款人），**截图保存**；
   - USDT：退回顾客**原始付款的钱包地址**（即你当初收款那笔的 from 地址），**记下退款交易哈希**。
4. **把退款凭证交给平台**（经客服/管理员录入后台「退款留痕」）：
   - 人民币：上传**退款截图**；
   - USDT：提供**退款交易哈希**，平台会**链上核验**该笔退款的收款地址是否==原付款地址、金额是否≥应退额。
5. 平台留痕该笔退款（含原路锁定地址、链上校验结果），**留存 ≥5年**。

> ⚠️ **务必原路、务必留证**：USDT 退到非原始地址、或金额/地址对不上，链上核验会判**失败**；人民币退给非原付款人属**退第三方**（反洗钱红线）。超出平台限额的退款会先转平台审核、放行后再退。如有疑问先联系平台客服，不要自行退给第三方账户。

> 🔒 **顾客隐私（脱敏）·〔2026-06-18 上线〕**：为保护顾客隐私，订单详情里顾客的**邮箱与手机号已部分打码**（如 `nezh****t2026@gmail.com` / `+374****0001`），且**手机号不可直接点击拨打**。**退款或其他事宜如需联系顾客，请用商家后台左侧「聊天」功能直接联系顾客**——站内聊天即可触达，无需手机号。

### 直付单被平台取消/退款后：商家端「待退款」标记流程 〔2026-06-17 上线；2026-06-18 扩展〕

当一笔直付订单进入「待退款」后（两种来源：① 平台管理员取消/退款你已确认收过款的单；② 〔2026-06-18 新增〕**顾客在 App 自助取消了一笔他已经付款给你（已上传付款凭证）的待确认单**——此时钱已到你账户、订单却被取消，需你原路退还），都会进入下面的流程：

1. 你会收到一条推送：「有一笔直付单需要您退款」。
2. 打开**商家后台 → 订单 → 待退款**（侧栏新增「待退款」角标），或直接打开该订单详情，会看到一张**「待您退款」卡**，上面有：
   - **应退金额**（≤原订单）、**原付款渠道**（微信/支付宝 或 USDT）；
   - USDT 单还会显示**须退回的原地址**（系统由原始付款交易反查锁定，只能退这个地址）；
   - **顾客邮箱 / 配送电话**，以及上方的**付款凭证截图**（付款人可对照）。
3. 你用**本人账户、按原路**把钱退还给原付款人（USDT 退回原地址；微信/支付宝退回原付款账户，个人收款码不支持原路退回时，按凭证里的账号重新转账退回）。
4. 退完后回到该卡点**「标记已退款」**：USDT 可选填**退款交易哈希**（便于平台链上核对），可填备注。确认后该单转为「已退款」留痕。

> ⚠️ 平台**全程不经手这笔钱**——是你在自己的账户里退给顾客；「标记已退款」只是把退款事实记录在平台、便于核对与留痕。原路、留证的红线同上。

### ⏰ 逾期未退款会被自动暂停接单〔2026-06-21 · 2026-06-22 改自动〕
平台取消/退款一笔直付单后，请你**尽快用本人账户把钱原路退还给顾客**，并到**商家后台 → 订单 → 待退款**把该单点**「标记已退款」**。
- **逾期不退的后果（非资金约束）**：超过平台设定的时间（默认 4 小时）仍未退款且未标记，系统会**自动催办你（邮件）+ 计入你的风控档案 + 告警平台运营**；继续逾期到平台设定的停接单时长（默认 **36 小时**），系统会**自动暂停你的接单**（只暂停接「新」单，不影响你已接的订单；停接单与你的保证金互相独立、不会从保证金扣钱）。
- **怎么恢复接单？** 你**用本人账户把钱原路退还顾客、并到「订单 → 待退款」点「标记已退款」后，系统会自动恢复你的接单**（前提是你没有别的逾期未退留痕），无需联系平台、也无需等待人工。
- **已经退了但忘标记？** 请尽快补点「标记已退款」即自动恢复；只要你已实际原路退款，也可联系平台客服核实解除。**平台不会替你退款、也不会从你的保证金扣钱赔顾客**——退款始终是你在自己账户原路退，平台只做记录与核验。

## 钱包（侧栏「私域资金池」）— 说明

哪吒采用"**顾客直付商家本人账户**"模式，平台**不向商家打款**。因此：

1. **钱包没有"提现/打款"功能**：原系统残留的「提现申请 / 请求提现 / 待提现 / 已提现总额 / 可提现余额 / 下次打款(Next Payouts)」等入口**已隐藏**，你不会也不需要在平台申请提现——你的营业款由顾客直接付到你自己的收款账户。 〔2026-06-17〕侧边栏「**报表**」里的「**打款报表**」「**收入报表**」也已隐藏——同理：平台不向你打款、直付订单不计入平台口径的"收入"，这两张报表恒空，留着只会让你误以为"系统坏了"。你的销售情况请看「订单报表 / 商品报表」。
2. **钱包页当前展示**：
   - **手持现金**：以现金/线下直收方式收到、需与平台结算口径对账的金额（App 内已停用货到付款/COD，此项通常为 0）；
   - **待付余额（应付平台）**：你需要支付给平台的金额（如佣金；扣佣机制上线后启用）。金额大于 0 时，会出现「立即支付 / Pay Now」入口供你向平台付款；
   - **累计佣金**：历史佣金统计（直付订单暂不计入）；
   - **支付记录**：你向平台付款的历史明细。
3. 平台仅靠**佣金**收费（从你预存的**预存佣金**余额中按单扣除），不经手你的营业资金。

> 说明：扣佣/充值台账等功能仍在建设中，上线后本节会同步更新具体操作步骤。

## 菜价与收款币种（德拉姆 ֏ 计价 · 顾客付人民币/USDT）

1. **你按德拉姆（֏）录菜价**：后台所有菜价、金额都以亚美尼亚德拉姆 ֏ 为准（例：一份兰州拉面填 `1500` 即 ֏1500）。

|---|---|---|
| ✅ **来源干净**（绝大多数） | 正常通过，无感知 | 照常确认收款、出餐 |
| 🚫 **来源命中制裁名单** | **自动拒收该单**，提示「该订单付款来源地址命中制裁名单，已自动拒收，请勿出餐并联系平台。」 | **不要出餐**；若这笔 U 已到账，**先别动用**，联系平台按"原路退回原地址"处理 |

### 三、制裁规避（亚美尼亚特有）—— 硬禁
亚美尼亚是对俄制裁规避的高风险地区，以下坚决不接：
- 帮俄罗斯转口采购 / 转运物流 / 代付转账
- 与受制裁实体 / 个人关联的业务

> ✅ **可以上的**：正规餐饮、超市零食、租房、二手闲置、包车出行、本地旅游、美容美发、正规按摩、正规移民 / 签证咨询等。其中移民 / 签证 / 按摩 / 医疗类属「需牌照 / 人工审」，平台会逐个人工核验后才放行。
>
> 系统已对发布内容做关键词自动筛查，命中上述硬禁业务词会被直接拒绝。如你确属**正规持牌**业务被误拦，请联系平台客服说明。

## 平台客服翻译功能（告知商家）
平台在线客服「小哪」已接入翻译功能：当骑手/配送员与顾客语言不通时，**可以让顾客把骑手发来的话（亚美尼亚语/俄语/英语）复制粘贴到客服对话框**，小哪会翻成中文并说明意思；顾客想回复的话也能翻成当地语言（亚美尼亚语+俄语，附中文回译核对）让顾客直接复制发给骑手。商家遇到顾客因语言不通沟通困难时，可提示顾客使用此功能。
