## 0. 怎么用 + 覆盖深度纪律（铁律）
1. **先摊全维度再跑**：18 维度逐个过，**不静默只跑 happy path**。
2. **覆盖深度诚实**：每维度标 ✅真机实测 / 🔶仿真 / ⬜未覆盖 / 👤需真人。**仿真≠实测、proxy≠真实路径、wired≠命中分支、单点≠矩阵**。
3. **🔴 硬门(must)全过(或真人签字)才可上线**；其余 gate 项有缺口要列清单 + 风险。
4. **数字/计数类必真数据不注入**；**≥2处显示的同一数字必同源或对账**（横幅幽灵数教训）。

## 2. 执行顺序（建议）
1. 先 **🔴硬门测试层**：第3 安全 → 第4 资金合规（出事不可逆/违法风险最高）。
2. 再 **第10 业务流程交叉验证**（含轴I不变量、登录/未登录、四类路径）——这是"全平台真实运作"核心。
3. 再 第5闭环 / 第1产品 / 第6性能 / 第9 i18n / 第8兼容。
4. **第7 容灾恢复演练**（上线前必补，唯一"出事不可逆"盲区）。
5. **C 上线就绪**(11演示数据清除→12真实商家→13开关态)：这些是**上线开关动作**，破坏性/真实影响，**逐个用户拍板**。
6. **D 人工门**：列清单交用户找律师/渗透。

## 3. go / no-go 判定
- **可上线** = 全部 🔴硬门 PASS 或真人签字(16/17/18) + gate 项缺口已知且风险可接受。
- **任一 🔴硬门未过 = no-go**。尤其：演示数据没清(11)、真实商家下不了单(12)、开关态没核(13)、备份没演练过恢复(7)、律师没审(16/18)。

## 4. 输出格式
1. 18 维度逐项：覆盖深度 + 证据 + go/no-go。
2. 第10层填满的交叉验证矩阵 + 轴I不变量报告。
3. **给用户亲手补验/拍板**清单：开关上线态、演示数据清除、律师、渗透、真链、真机。
