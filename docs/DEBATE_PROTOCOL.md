# /debate 对抗核验协议 v2（回声室修正版）

用途：高风险动作（资金机制/L1 毗邻/上线部署/大改前端/两次修不好的顽固 bug）动手前的多实例对抗核验。
v1 病灶：同上下文+同角色的多实例会收敛到共识而非真相；辩"哪个方案好"常在错误的问题上精细化。
出处：fable-brief/PLAN_opus_uplift.md（P4，2026-07-03 Fable 起草，2026-07-04 业主批准落地）。

## 第 0 轮 · 先辩问题（10 分钟封顶）

辩题不是"方案 A 还是 B"。各方先独立回答：
1. 我们在解的问题，一句话表述是什么？它是根因还是症状？
2. 换一层看（写侧/读侧、机制/呈现、预防/兜底），问题还成立吗？
3. 有没有"不做"的选项？

任一实例给出更好的问题表述 → 用新辩题重开。多数僵局死在这一步没做。

## 角色与证据面分割（防回声室核心）

发起人分配角色，各方**先独立成稿再互看**：
- **A 代码侦探**：只准引用真实代码/git log/DB 实况；禁引用竞品与"用户会喜欢"
- **B 用户代言人**：只站真实使用者处境（几步走完/看得懂吗/打扰吗）；禁引用实现成本
- **C 风险审计**：只读 INVARIANTS/docs/compliance/回滚路径，输出"最坏情况清单"

（两实例版 = A+C 合并；单人自辩至少把三顶帽子分三段戴。）

## 每方开场必交"可证伪判据"

格式："如果 ___（具体证据/实验结果）出现，我的主张就错了。"
- 给不出判据 = 该主张是偏好不是论断，权重降级
- 三方都给不出 = 辩题失格，回第 0 轮
- 判据能当场测的当场测（读码/跑脚本/查线上）；测出来的证据 > 修辞

## 终止规则（防无限辩）

- 两轮无新证据（只有措辞重排）= 回声室已成 → 停，三选一：换框重开 / 交 Fable 裁决 / 缩小赌注灰度试
- 出现"双方都对但前提不同" → 把前提差写成开关/分支，不强行分胜负

## 裁决输出模板（发起人/裁判填）

1. 采纳案 + 关键证据（不是关键修辞）
2. 败案的残余风险 → 转成监控项/回滚触发条件（败方观点不丢弃，变成告警）
3. 可证伪判据留档：上线后什么信号出现 = 本裁决要重审
