## 3. 真实通知签收

### 3.1 证据口径

“已入队”“provider HTTP 2xx”“配置开启”“站内信已写库”都不等于真实收达。每条真实回执至少记录：业务事件 ID、目标角色、遮蔽收件端点、触发时间、入队/worker/provider message ID、provider 接受时间、设备实际展示时间、前台/后台/锁屏状态、点击落点、文案/金额/订单号、重复数、截图/录像。禁止记录 token、完整手机号、完整邮箱、完整 chat ID 或真实秘密。

在场感知必须一并验证：接收方正打开对应会话/订单时，冗余提醒按产品规则被抑制或诚实更新；不在场才投递并产生真实未读；查看后真实清除。不得用假红点、假未读或测试脚本生成的“成功”文案代替外部回执。
