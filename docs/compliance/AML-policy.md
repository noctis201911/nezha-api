# 哪吒外卖 — 反洗钱与制裁合规政策 (AML / CTF Policy)

**版本**: 1.0 　**生效日期**: 2026-06-12 　**适用主体**: 哪吒外卖平台 (nezha.am, 亚美尼亚埃里温运营)

> 本文件为对外合规说明文档,供监管机构、银行、支付伙伴、法律顾问审阅。描述平台为防范洗钱、恐怖融资及制裁规避所建立的措施。文中区分**已实施**与**规划中**的控制项,如实陈述。

## 1. 平台业务模式概述
哪吒外卖是面向亚美尼亚埃里温华人社群的**外卖撮合平台**,连接当地餐馆(商家)与顾客。平台定位为**纯撮合 + 收佣的市场/SaaS**:
- 顾客付款**直接付进商家本人**的收款账户(人民币:微信/支付宝;USDT:链上地址),**平台全程不归集、不代收代付资金**。
- 平台收入来自向商家收取的佣金,通过商家预存佣金（履约保证金）按单扣除(合法 B2B)。
- 详见 `business-flow.md`。

此模式从结构上**避免了二次清算(无证经营支付结算)**,降低了平台成为资金归集/转移通道的洗钱风险。

## 2. 合规承诺
平台承诺遵守适用的反洗钱(AML)、反恐怖融资(CTF)及经济制裁法规,建立基于风险的交易监控与处置机制,对可疑交易留痕、审查、必要时拒绝。

## 3. 交易监控规则 (已实施 — 交易红旗风控①)
平台对每笔下单/支付意图执行自动风险评估,规则阈值集中存于后台配置项、可由管理员调整(非硬编码)。处置分三档:**拒单 / 转人工审核 / 放行**。

| 规则 | 触发条件(默认阈值) | 处置 |
|---|---|---|
| 单笔限额 | 法币 > 等值 100 美元;USDT > 200 美元 | **拒单**,提示联系客服 |
| 单账号单日累计 | 法币 > 300 美元;USDT > 500 美元 | 转**人工审核** |
| 交易频次 | > 5 单/24小时 或 > 2 单/10分钟 | 转**人工审核** |
| 金额特征 | 整百/整千金额、或显著偏离餐饮客单价的大额 | 标记**人工审核** |

- 命中"拒单"或"人工审核"的订单,**顾客在审核通过前无法取得商家收款码,即无法完成付款**("先拦付款"原则)。
- 人工审核由平台管理员在后台「风控审核队列」处理,可**放行 / 退款 / 清退**,每步留痕。
- 放行后该顾客在限定宽限期内可重新完成支付。

> 当前状态:风控引擎与后台已上线,总开关默认关闭(待菜单计价单位标准化后启用)。启用后即按上表执行。

## 4. 制裁与黑名单地址筛查 (规划中 — USDT通道风控②)
针对 USDT 收款通道,平台将实施:
- 对付款来源链上地址比对 **OFAC SDN 制裁名单**(公开数据)及已知高风险/黑名单地址库;
- **命中即拒收**该笔交易并记录;
- 后续可对接专业链上分析服务(Chainalysis / TRM / Elliptic)增强。

## 5. 链上交易记录留存 (规划中 — ②)
每笔 USDT 交易记录:交易哈希、来源地址、币种、金额、时间、对应订单号,**入库留存不少于 5 年**,以备合规审查。

## 6. 退款政策 (合规红线)
- 退款**仅允许原路退回**:原支付方式 / 原卡 / 原钱包 / 原 USDT 付款地址;
- 退款金额**不超过原订单**;
- **禁止退至任何第三方账户/钱包**;
- 因平台不持有资金,退款由商家按上述规则执行,平台系统对退款指令做"原路"锁定与留痕。

## 7. 可疑交易处理流程
1. 自动规则命中 → 写入风控记录(订单号、账号、命中规则、金额、时间、处置结果)+ 推入人工审核队列;
2. 管理员核实(可联系顾客)→ 放行 / 退款(原路) / 清退,操作留痕;
3. 所有记录长期保存,供审计。

## 8. 记录与留痕
- 风控命中记录表 `nezha_risk_records`:订单号、账号、支付通道、金额、命中规则、处置结果、处理人、处理时间。
- 链上记录(规划中)留存 ≥ 5 年。
- 全部通过 git 版本管理,变更可追溯。

## 9. 角色与职责
- **平台管理员**:监控审核队列、处置可疑交易、维护阈值与名单、保管记录。
- **商家**:如实提供本人收款信息,按原路规则执行退款。
