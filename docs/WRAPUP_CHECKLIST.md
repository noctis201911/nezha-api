# 哪吒收尾排查清单（每个任务做完、汇报前逐条过）

> 用途：每次改完功能、汇报"做好了"之前，**逐条**走一遍下面的清单。
> 由来：2026-06-14 商家后台 QA 收尾时，发现一批"用户没想到、我没主动查"的盲区，统一固化成规则。
> 读法：任何窗口可执行 `node nz.js run "cat /www/wwwroot/api.nezha.am/docs/收尾排查清单.md"` 读最新版。

## 0. 验证铁律（不过则不算做完）
- [ ] 前端/页面改动：Playwright 真机截图 + console **0 error**（HTTP 200 / build 成功 ≠ 验证）。
- [ ] 三态分开验：空态 / 假数据态 / 真实登录态。

## 1. 合规 / 资金红线（动这些必须先停下问用户）
- [ ] 没碰：平台不碰钱 / 退款只原路退 / USDT 原地址 / 链上留存≥5年 / 二清腿 / 制裁名单 / PII 加密删除。
- [ ] 改过合规相关 → 同步更新 `docs/compliance/` + `CHANGELOG.md`。

## 2. 上线真实影响（会不会误伤真实用户/订单）
- [ ] 新开关的**默认态**对不对？默认开还是关，会不会误伤真实顾客/商家。
- [ ] 破坏性操作（删除/覆盖/批量改）有没有先确认、能不能回滚。
- [ ] **货币 / 金额**：改币种只换"符号"不换"数字"。确认价格数字本身是按当前币种录的（人民币就按元/分逻辑，别把美元数额当人民币）。当前默认币种 = **CNY（¥）**，业务设置项 `currency`。

## 3. 安全 & 数据
- [ ] **临时脚本删干净**：往服务器推过的一次性 `*.php`（seed/probe/fix 之类）全部 `rm`，别留在项目根。
- [ ] webroot / 项目根没有可疑脚本（如 `reset-pwd.php`），密钥 / 密码 / token 没写进代码或 git。
- [ ] PII（电话/邮箱/地址/证件）没明文泄露到日志、前端、提交里。

## 4. 残留测试数据
- [ ] 自己建的测试数据（名字带 QA / 测试 / 删我 / test 的商品/加料/券/账号）跑完即删。
- [ ] 注意区分"残留垃圾"（删）和"测试环境"（如 QA 登录账号 `人工测试账号`、`测试商家店` —— 删前先问，别误删登录账号）。

## 5. 脏数据 vs 显示层（别只贴膏药）
- [ ] 显示对了不代表数据对了：先确认根因在"数据"还是"代码/拼装处"。例：`+86+86` 是 DB 存了双国码，显示层去重只是遮住，要记录根因/告知用户去源头改。

## 6. 翻译 / 文案残留
- [ ] 没有英文未翻译 / 机翻味 / 乱码 / 重复标题 / 残留旧控件。
- [ ] 翻译文件 `resources/lang/zh|en/messages.php` 默认归翻译窗口；要改先 `git status` 看有没有别窗口未提交改动，只改必要键、`git add` 精确文件。

## 7. 可维护 / 防误改
- [ ] 关键改动加了注释/防回归说明（防下个人或下个窗口再踩）。
- [ ] 后台功能有变动 → 同步更新 `ADMIN_GUIDE.md` 对应章节。

## 8. 可逆性
- [ ] 改后端文件前已 `cp <file> <file>.bak.<时间>` 备份；`php -l` 过语法。
- [ ] 数据/设置改动可回滚（记下原值）。

## 9. 多窗口协调（服务器单一工作目录，多窗口共享）
- [ ] **绝不 `git commit -am`**：先 `git status`，只 `git add <本任务文件>` 再 commit，避免扫进别窗口/翻译窗口的未提交改动。
- [ ] 每条 bug 验证通过后**立即**提交，不攒批。
- [ ] 发现别的窗口在改同一批东西 → 停下告知用户、协调分工，别对着干。

## 10. 列表 / 分页 / 多屏正确性（按"交互类型"查，不只按界面查）
> 由来：2026-06-16 首页"加载更多"复制同一批商家的 bug，四轮 QA（商家端/资金流/admin/顾客端 UI）都没抓到——因为它们都按"界面/链路"切，没人按"翻第二页对不对"切；且 bug 在首屏外、要多点一下、数据要 ≥2 页才暴露。
- [ ] 凡是分页 / 下拉加载 / "加载更多"的列表：先把种子数据灌到 **≥2 页**（如商家 >10 家）再测，别用一页装得下的小数据集糊弄过去。
- [ ] **滑到底加载第 2 屏**，断言第 2 屏 ≠ 第 1 屏（**无重复**，不是把上面那批复制下来）。
- [ ] 断言**到底末态**正确出现（"没有更多…"），且不是无限循环永远能加载。
- [ ] 截图必须含**滑到底那一帧**，不能只截页面顶部（`fullPage:false` 只截顶部会漏掉首屏外的 bug）。
- [ ] 可复用工具：本机 `verify_loadmore.js`（逐屏抓名称、比对重复数、检查末态）。
