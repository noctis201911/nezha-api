### 3.3 核验线下支付（最重要！每天必做）

![线下支付列表](https://api.nezha.am/admin-docs/screenshots/06_orders_offline_payment.png)

**什么是线下支付？**  
顾客选择用人民币转账或USDT转账付款。顾客下单后，需要自己转账，然后上传转账截图。**你必须确认收到款，商家才会开始出餐。**

**核验步骤：**

```
第一步：点左侧菜单「订单管理」→「线下支付」
第二步：你会看到「线下汇款流水核销」页面
       - 全维视野（所有）
       - 待验证（还没确认的）← 重点看这里
       - 支付已验证订单（已确认的）
       - 验证被拒订单（拒绝的）

第三步：点「待验证」，找到需要处理的订单
第四步：点订单号，进入订单详情

第五步：在右侧「支付核验」区域
       - 查看顾客上传的转账截图
       - 确认金额正确
       - 确认转账到你指定的收款账号
       
第六步：确认无误 → 点「验证支付」按钮
       → 订单状态自动更新，商家开始出餐
```

⚠️ **如果发现转账有问题（金额不对/转错账号）：**  
点「拒绝」按钮，顾客会收到通知，需要重新上传。

### 3.4 退款处理

![退款列表](https://api.nezha.am/admin-docs/screenshots/08_order_refund.png)

当顾客申请退款时，会出现在这里。

**步骤：**
```
1. 点左侧菜单「订单管理」→「订单退款」→「新退款申请」
2. 查看退款原因
3. 判断是否同意退款：
   - 同意 → 点「批准退款」→ 款项退回顾客钱包

```

> 💡 **重要：** 退款只能退到顾客的平台钱包余额，不能直接退到银行卡/转账账号。

## 4. 调度中心——骑手派单

![调度中心](https://api.nezha.am/admin-docs/screenshots/09_dispatch_searching.png)

**什么时候用这里？**  
当订单商家已出餐，但还没有骑手接单时，这里会有记录。你可以手动指派骑手。

**两个分类：**

| 分类 | 意思 |
|------|------|
| 寻找骑手中 | 正在等待骑手接单的订单（数字=待派单数量） |
| 进行中订单 | 骑手已接单、正在送餐的订单 |

**手动派单步骤：**
```
1. 点「寻找骑手中」
2. 找到对应订单，点右侧「分配骑手」
3. 从骑手列表中选择一个在线的骑手
4. 点确认 → 骑手会收到接单通知
```

## 5. 风控中心——拦截可疑订单

风控中心是平台的安全保障，会自动拦截金额过大、下单过于频繁等可疑订单。

### 5.1 审核队列

![风控审核队列](https://api.nezha.am/admin-docs/screenshots/11_risk_queue.png)

当有订单触发风控规则时，会出现在这里，顾客付款后会被暂时**冻结**，等你人工审核。

**队列里每列的意思：**

| 列名 | 意思 |
|------|------|
| 时间 | 订单下单时间 |
| 顾客 | 顾客账号 |
| 餐馆 | 哪家餐厅 |
| 通道 | 用什么方式付款（人民币/USDT） |
| 金额 | 订单金额 |
| 命中规则 | 触发了哪条风控规则（如：金额超限/高频下单） |
| 处置 | 放行 或 拒绝 |

**怎么处理：**
```
看是真实顾客的正常大额订单 → 点「放行」→ 订单正常处理
看起来是可疑/欺诈订单 → 点「拒绝」→ 订单取消，款项原路退回
```

### 5.2 风控日志

![风控日志](https://api.nezha.am/admin-docs/screenshots/12_risk_logs.png)

这里记录了所有被风控系统处理过的历史记录，方便事后查证。

### 5.3 风控设置

![风控设置](https://api.nezha.am/admin-docs/screenshots/13_risk_settings.png)

可以调整风控的拦截阈值，例如：
- 单笔订单上限金额（超过就拦截）
- 24小时内同一账号最多下几单
- 大额特征金额标记

> ⚠️ **修改风控设置前请谨慎！** 改太宽松会放过可疑订单，改太严格会误拦正常顾客。

**当前设置（2026-06-14）：**  
风控总开关处于「禁用」状态（不拦截任何订单）。  
→ 上线后建议开启，先设置较宽松的阈值再逐步收紧。

![新入驻申请](https://api.nezha.am/admin-docs/screenshots/18_restaurant_pending.png)

```
1. 点左侧「商家管理」→「新入驻申请」
2. 查看商家提交的信息（餐厅名称、地址、负责人联系方式等）

4. 认为有问题 → 点「拒绝」（可填原因）
```

### 6.3 查看商家详情

![商家详情](https://api.nezha.am/admin-docs/screenshots/restaurant_detail.png)

在商家列表点👁️进入商家详情，可以看到：

**顶部标签栏：**

| 标签 | 内容 |
|------|------|
| 全局概览 | 该商家的收入、余额、待结算金额 |
| 订单 | 该商家的历史订单 |
| 商品 | 该商家的所有菜品 |
| 评价 | 顾客对该商家的评价 |
| 营销减免金额 | 活动/优惠券减免了多少 |
| 明细表 | 详细流水 |
| 设置 | 修改商家基本设置 |
| 收款信息 | 该商家的收款账号（人民币/USDT） |
| 通讯 | 与商家的消息往来 |
| 商业套餐 | 该商家适用的佣金比例 |
| 结算记录 | 历史结算记录 |

**商家资金状态（重要）：**
- **商家待结算金额**：顾客已付款但平台还没给商家结算的金额
- **已派发资金总量**：历史上平台已经给该商家结算过的总金额
- **余额**：商家钱包余额
- **累计佣金**：平台从该商家赚到的总佣金

**"发起强制扣款"按钮：** 当商家欠平台钱（如退款后佣金为负）时，可强制从商家余额扣款。

> 📌 **重要：** 哪吒外卖采用"商家直接向顾客收款"的模式，平台不经手顾客的钱。结算是指平台将应给商家的部分（顾客已支付减去平台佣金）和骑手工资定期发放。

### 11.1 商家结算

![商家结算](https://api.nezha.am/admin-docs/screenshots/44_restaurant_disbursement.png)

**每期结算步骤：**
```
1. 点左侧「结算管理」→「商家结算」
2. 查看待结算商家列表和金额
3. 确认金额无误
4. 点「结算」→ 系统记录结算完成
   （实际转账需要你在银行/USDT钱包手动操作）
5. 结算后在系统里标记为"已结算"
```

### 11.2 骑手结算

![骑手结算](https://api.nezha.am/admin-docs/screenshots/45_dm_disbursement.png)

## 13. 销售终端（POS）——线下收银

![POS界面](https://api.nezha.am/admin-docs/screenshots/72_pos.png)

POS系统用于在店内帮顾客手动下单（无需顾客自己用App）。

**使用步骤：**
```
1. 点左侧「销售终端」
2. 左侧「商品版块」：
   - 先选「配送区域」
   - 再选「商家」
   - 选择「分类」浏览菜品
   - 点击菜品加入右侧账单

3. 右侧「账单区块」：
   - 选择顾客（搜索已注册顾客）或点「添加新顾客」
   - 选择订单类型（自取/平台配送）
   - 确认金额后点「提交订单」
```

## 附录：常见问题

**Q：顾客说下单了但商家没收到怎么办？**  
A：去订单管理→全部订单→搜索该顾客订单，查看订单状态。如果显示"待处理"，说明商家还没接单，可以联系商家。

**Q：顾客转账了但我这边收不到怎么办？**  
A：先检查转账是否到账（检查你的银行/USDT钱包），再去「线下支付」查看该笔订单，顾客是否上传了转账凭证。

**Q：想修改平台的抽成比例怎么办？**  
A：在「业务配置」→「基本信息」里修改佣金率，或在具体商家详情的「商业套餐」里单独为某家商家设置不同的比例。

**Q：系统显示某个商家"未命中有效菜系"怎么办？**  
A：进入该商家详情→设置，重新为商家选择正确的菜系（Cuisine）类型，保存后即可解决。
